Guía de Postman para consumir SWAPI

Introducción

Postman es una herramienta que te permite interactuar con APIs de manera sencilla. En esta guía, aprenderás a usar Postman para consumir la API de Star Wars (SWAPI) y realizar tus primeras solicitudes HTTP.

Requisitos:

  • Descargar e instalar Postman desde aquí.
  • Tener una conexión a Internet.

Las colecciones en Postman te permiten organizar tus solicitudes en grupos. Esto es útil cuando trabajas con varias APIs o endpoints.

  1. Abre Postman y selecciona el ícono de Collections en la barra lateral.
  2. Haz clic en New Collection y asigna un nombre, por ejemplo, "SWAPI Collection".
  3. Guarda la colección.
Tip: Usa nombres descriptivos para tus colecciones, como "Pruebas SWAPI" o "API de GitHub".

Las variables te permiten reutilizar valores en varias solicitudes. En este caso, definiremos una variable para la URL base de SWAPI.

  1. En la colección, haz clic en los tres puntos y selecciona Edit.
  2. Ve a la pestaña Variables.
  3. Añade una variable llamada base_url con el valor https://swapi.dev/api.
  4. Guarda los cambios.
Nota: Asegúrate de que la URL esté correctamente escrita. Un error común es olvidar la barra final (/) en la URL.

Crea solicitudes GET para cada uno de los siguientes recursos:

  • Personajes: {{base_url}}/people/
  • Películas: {{base_url}}/films/
  • Planetas: {{base_url}}/planets/
  • Especies: {{base_url}}/species/
  • Naves espaciales: {{base_url}}/starships/
  • Vehículos: {{base_url}}/vehicles/
Ejemplo: Al hacer una solicitud a {{base_url}}/people/1/, obtendrás información sobre Luke Skywalker.

Para obtener información específica, puedes realizar peticiones GET con un ID:

  • Personaje por ID: {{base_url}}/people/{{person_id}}/
  • Película por ID: {{base_url}}/films/{{film_id}}/
  • Planeta por ID: {{base_url}}/planets/{{planet_id}}/
Práctica: Cambia el valor de {{person_id}} por un número del 1 al 10 y observa los resultados.
