💻 Sesiones en PHP

Guía profesional para alumnado de FP DAW. Incluye enlaces a documentación oficial y práctica con GitHub.

1️⃣ Introducción

En esta práctica aprenderás a gestionar sesiones en PHP para mantener información entre diferentes peticiones HTTP. Este mecanismo permite recordar usuarios, preferencias y estados de navegación.

PHP ofrece una API sencilla para trabajar con sesiones, descrita en la documentación oficial: Manual oficial de PHP sobre sesiones.

2️⃣ Conceptos esenciales

  • 🔹 HTTP es un protocolo sin estado: cada petición es independiente.
  • 🔹 Sesiones: almacenan información del usuario en el servidor, identificada por un ID de sesión (cookie o URL).
  • 🔹 Variables de sesión: disponibles en el array superglobal $_SESSION.
  • 🔹 Inicio y destrucción: session_start() y session_destroy().
  • 🔹 Configuración: mediante php.ini o funciones como session_set_cookie_params().

Consulta la sección Ejemplos básicos de sesión en PHP para ver usos simples y seguros.

3️⃣ Buenas prácticas

💡 Las sesiones deben gestionarse de forma segura, especialmente en aplicaciones con autenticación o datos sensibles.

  • ⚙️ Llamar siempre a session_start() al inicio del script.
  • 🔒 Regenerar el ID de sesión tras el inicio de sesión con session_regenerate_id(true).
  • 🚫 Evitar almacenar contraseñas u objetos grandes en la sesión.
  • 🧹 Destruir correctamente la sesión al cerrar sesión (ver session_destroy()).
  • 🧠 Mantener el estado mínimo necesario: identificador, nombre, rol, etc.

4️⃣ Actividad práctica

Esta práctica consiste en crear una aplicación básica que utilice sesiones para gestionar el acceso de usuarios.

Paso 1 · Preparar el entorno
  1. Accede al repositorio base: profeInformatica101/app_php.
  2. Haz clic en Fork para crear tu copia personal en tu cuenta.
  3. Clona tu repositorio: git clone https://github.com/tu_usuario/app_php.git.
Paso 2 · Objetivo del ejercicio

Implementar un sistema mínimo de autenticación con sesiones, donde el usuario pueda iniciar y cerrar sesión. Debe mostrarse un mensaje personalizado cuando la sesión esté activa.
