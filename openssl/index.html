Tutorial OpenSSL

OpenSSL es una biblioteca de software de código abierto utilizada para el cifrado y la comunicación segura en redes. Este tutorial muestra cómo usar OpenSSL para intercambiar un archivo entre dos partes. Se divide en 3 secciones: cifrado asimétrico, certificados y RSA. Si ya tienes conocimientos básicos, puedes saltar directamente a la sección 5.

Alice y Bob desean comunicarse de manera segura en una red insegura, evitando que Eve intercepte y comprenda los mensajes. Usan claves públicas y privadas para asegurar confidencialidad y autenticidad. Alice puede cifrar un mensaje con la clave pública de Bob, y Bob lo descifra con su clave privada. También, Alice puede firmar un mensaje con su clave privada para que Bob verifique su autenticidad.

Los certificados son usados para garantizar que las claves públicas son auténticas. Alice y Bob generan solicitudes de firma de certificado (CSR) que son firmadas por una Autoridad Certificadora (CA). Esto crea una "cadena de confianza" que Alice y Bob pueden usar para verificar sus certificados.

RSA se basa en seleccionar dos números primos grandes, calcular su producto (modulus) y utilizar claves públicas y privadas para cifrar y descifrar mensajes. RSA es eficaz para el intercambio de claves simétricas, pero no para cifrar grandes volúmenes de datos directamente.

Este protocolo combina el uso de cifrado asimétrico, certificados y RSA para el intercambio seguro de mensajes. Alice genera un certificado, cifra una clave simétrica con la clave pública de Bob y firma la clave con su clave privada. Bob verifica la firma y usa la clave simétrica para descifrar mensajes grandes.

Este ejemplo muestra comandos para generar claves, firmar certificados y cifrar datos usando OpenSSL. Puedes crear carpetas separadas para Alice, Bob y la CA, siguiendo los pasos detallados para generar claves privadas, públicas y certificados, así como para intercambiar y verificar mensajes cifrados.
