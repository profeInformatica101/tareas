Solución al Problema de Maven

"No compiler is provided in this environment. Perhaps you are running on a JRE rather than a JDK?"

Descripción del Problema

Este error ocurre cuando Maven no encuentra el compilador de Java porque el entorno está utilizando una JRE en lugar de una JDK.

Pasos para Resolver el Problema

  1. Verifica que tienes instalada la JDK:

    Ejecuta el siguiente comando en la terminal:

    javac -version

    Si el comando no muestra una versión de javac, instala la JDK. Por ejemplo, en Ubuntu o Debian:

    sudo apt install openjdk-17-jdk

    En Fedora o CentOS:

    sudo dnf install java-17-openjdk-devel
  2. Configura la variable de entorno JAVA_HOME:

    Establece JAVA_HOME a la ubicación de la JDK en tu sistema.

    En Windows:

    • Abre "Propiedades del Sistema" y accede a "Variables de entorno".
    • Agrega una nueva variable de sistema llamada JAVA_HOME con el valor de la ruta a tu JDK.
    • Actualiza la variable PATH para incluir %JAVA_HOME%\bin.

    En Linux o macOS:

    Agrega las siguientes líneas al archivo ~/.bashrc, ~/.bash_profile o ~/.zshrc (dependiendo de tu shell):

    export JAVA_HOME=$(dirname $(dirname $(readlink -f $(which javac))))
export PATH=$JAVA_HOME/bin:$PATH

    Después, recarga el archivo de configuración con:

    source ~/.bashrc
  3. Verifica la configuración de Maven:

    Ejecuta mvn -v y verifica que Java home apunte a la JDK en lugar de una JRE.

  4. Reinicia el entorno de desarrollo:

    Si estás usando un IDE, asegúrate de que esté configurado para usar la JDK en lugar de la JRE en la configuración de Java del IDE.

Resumen

Para solucionar este problema:

  • Asegúrate de tener una JDK instalada.
  • Configura JAVA_HOME para que apunte a la JDK.
  • Verifica que Maven esté usando la JDK configurada.

Después de realizar estos pasos, el problema debería resolverse y Maven podrá compilar el proyecto correctamente.
