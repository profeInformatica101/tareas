Tareas para Probar el Debugger

Código Java con Error 
package com.ejemplos.debugger;

public class Ejemplo2 {
    public static void main(String[] args) {
        int n = 5;
        int[] tabla = new int[n];
        
        tabla = llenar(n);
        int suma = sumar(tabla);
        
        System.out.println("La suma es: " + suma);
    }

    private static int[] llenar(int n) {
        int[] tabla = new int[3]; // Error: tamaño fijo de 3 en lugar de usar 'n'
        for (int i = 0; i < n; i++) { // Esto causará un error de índice fuera de rango
            tabla[i] = i * 10;
        }
        return tabla;
    }

    private static int sumar(int[] tabla) {
        int suma = 0;
        int n = tabla.length;
        for (int i = 0; i < n; i++) 
            suma = suma + tabla[i];
        return suma;
    }
}
1. Identificar el error en el método llenar
  • Establece un breakpoint en la línea donde se declara int[] tabla = new int[3].
  • Observa que el tamaño del array no corresponde con el valor de n.
2. Depurar el bucle para verificar el error de índice
  • Establece un breakpoint dentro del bucle for del método llenar.
  • Observa cómo i excede el tamaño del array, causando un error de índice fuera de rango.
3. Evaluar expresiones durante la depuración
  • Establece un breakpoint en la línea donde se inicializa tabla[i].
  • Usa la funcionalidad Evaluate Expression y verifica los siguientes valores:
    • tabla.length (debería ser 3 en lugar de 5).
    • n (valor inicial de 5).
    • i (cuando excede el rango).
4. Corregir el tamaño del array y validar
  • Corrige el tamaño del array en el método llenar usando int[] tabla = new int[n].
  • Vuelve a ejecutar el programa en modo depuración y valida el correcto funcionamiento.

Objetivo General

Este ejercicio enseña a los estudiantes a:

  • Identificar errores de tamaño en arrays.
  • Inspeccionar variables y detectar errores de índice fuera de rango.
  • Evaluar expresiones clave durante la depuración para validar hipótesis.
