Depuración en Java para Principiantes

Guía paso a paso para aprender a encontrar y corregir errores (“bugs”) en tus primeros programas Java, usando desde simples mensajes en consola hasta herramientas de depuración.

Pensado para estudiantes que están comenzando a programar en Java y que usan un IDE (Eclipse, IntelliJ, NetBeans) o la línea de comandos.

Java básico Depuración Aprendizaje Principiantes

¿Para quién es este tutorial?

  • Estudiantes que han escrito ya sus primeros programas Java (bucles, condicionales, métodos).
  • Personas que han visto errores en pantalla y no saben bien cómo interpretarlos.
  • Alumnado que quiere aprender a usar el depurador del IDE y no solo System.out.println().

1. Introducción: ¿Qué es depurar?

Idea clave: Depurar es entender qué está pasando realmente en tu programa cuando no hace lo que tú esperas, para poder corregirlo.

Cuando empiezas a programar es muy normal que los programas tengan errores. A estos errores los llamamos bugs. Depurar es el proceso de:

  1. Reproducir el error (hacer que vuelva a ocurrir).
  2. Investigar en qué parte del código se produce.
  3. Entender por qué ocurre.
  4. Modificar el código para corregirlo.

En Java, podemos depurar de varias formas, desde las más simples (mensajes por pantalla) hasta herramientas completas de depuración incluidas en los IDEs.

Tipos de depuradores en Java

  • Depurador del IDE (Eclipse, IntelliJ, NetBeans): lo usaremos mucho en clase.
  • Herramientas en consola: como jdb, útil para entender qué pasa “por dentro”.
  • Mensajes en consola: el clásico System.out.println().

2. Antes de depurar: entender el error

2.1 Tipos de errores básicos

  • Errores de compilación: el programa ni siquiera se ejecuta.
    Ejemplo: te falta un ;, un }, o usas una variable no declarada.
  • Errores de ejecución: el programa arranca, pero se para de golpe con una excepción.
    Ejemplo: NullPointerException, ArrayIndexOutOfBoundsException.
  • Errores lógicos: el programa no se rompe, pero el resultado no es el esperado.
    Ejemplo: calculas mal un descuento o una media.

2.2 Leer el mensaje de error

Cuando Java muestra un error, casi nunca es “ruido”; normalmente te dice:

  • Tipo de error (por ejemplo, NullPointerException).
  • Línea del código donde ha ocurrido.
  • A veces, el método y la clase que estaban en uso cuando ocurrió el error.
Exception in thread "main" java.lang.ArithmeticException: / by zero at Calculadora.dividir(Calculadora.java:10) at Calculadora.main(Calculadora.java:20)

Aquí podemos ver que el problema está en la línea 10 de Calculadora.java, dentro del método dividir, porque estamos dividiendo entre 0.

3. Conceptos básicos de depuración

Pensar como detective: El depurador te ayuda, pero quien resuelve el caso eres tú. Necesitas hacer preguntas al código: “¿Qué valor tenía esta variable?”, “¿Ha entrado en este if?”, “¿Cuántas veces se repite este bucle?”.

3.1 Depurar con mensajes en consola

Es la forma más sencilla y muy útil cuando empiezas. Consiste en ir escribiendo mensajes para ver por dónde pasa el programa y qué valores tienen las variables.

public static int sumarHasta(int n) { int suma = 0; System.out.println("Inicio del método sumarHasta con n = " + n); for (int i = 1; i <= n; i++) { suma += i; System.out.println("i = " + i + " | suma = " + suma); } System.out.println("Fin del método sumarHasta. Resultado = " + suma); return suma; }

Con estos mensajes puedes ver si el bucle se repite el número de veces correcto y si la variable suma acumula bien los valores.

3.2 Puntos de ruptura (breakpoints)

Un breakpoint es como decirle al programa: “para aquí y deja que mire qué está pasando”.

  • Se marcan en el IDE haciendo clic en el margen de la línea.
  • Al ejecutar en modo “depuración”, el programa se detiene en esa línea.
  • Desde ahí puedes avanzar paso a paso, ver variables, etc.

3.3 Ejecución paso a paso (step)

  • Step over: ejecuta la línea actual y pasa a la siguiente.
  • Step into: si la línea llama a un método, entra dentro de ese método.
  • Step return: ejecuta hasta que termina el método actual.

3.4 Trazas de pila (stack trace)

La traza de pila es “el camino” de métodos que se han ido llamando hasta llegar al error.

try { metodoA(); } catch (Exception e) { e.printStackTrace(); // Muestra la traza completa }

Leyendo la traza puedes saber qué métodos se han ejecutado y en qué orden, hasta que algo ha fallado.

4. Depuración en aplicaciones cliente sencillas

En el nivel inicial solemos trabajar con programas de consola o pequeñas interfaces gráficas. Las ideas clave son las mismas:

4.1 Ejemplo: programa que calcula el IMC

public static void main(String[] args) { double peso = 70; double altura = 0; // ¡Error! No podemos dividir entre 0 double imc = peso / (altura * altura); System.out.println("Tu IMC es: " + imc); }

Si ejecutas este código, se producirá una ArithmeticException. Para depurarlo, puedes:

  • Leer el mensaje de error: te dirá que divides entre 0.
  • Poner un breakpoint en la línea del cálculo y ver el valor de altura.
  • Corregir el código comprobando que altura no sea 0.

4.2 Técnicas rápidas que puedes usar ya

  • Imágenes mentales: dibuja en papel qué valores deberían tener las variables en cada paso.
  • Poner nombre claro a las variables: notaMedia es mejor que x.
  • Dividir el código en métodos pequeños: es más fácil depurar un método de 10 líneas que uno de 200.

5. Depuración en el lado servidor (vista muy básica)

En este nivel, basta con entender la idea general. Más adelante en el curso veremos servlets, JSP y aplicaciones web en detalle.

En aplicaciones web Java (por ejemplo, con servlets o JSP), el código no se ejecuta en tu ordenador, sino en un servidor. La idea de depurar es la misma, pero:

  • Puede haber muchos usuarios usando la aplicación al mismo tiempo.
  • El servidor crea hilos para atender las peticiones.
  • A veces depuramos “a distancia” (depuración remota).

5.1 Técnicas sencillas en servidor

  • Escribir mensajes en un fichero de log del servidor.
  • Añadir parámetros de depuración en la URL, por ejemplo ?debug=true.
  • Probar primero la lógica en una clase con main (sin servidor) y depurarla ahí.

6. Depuración de aplicaciones multihilo (idea introductoria)

Hilo = tarea paralela. De momento, quédate con la idea: varios hilos pueden intentar usar la misma variable al mismo tiempo, y eso puede producir errores “raros”.

Aunque en tu primer contacto con Java normalmente usarás un solo hilo, es útil saber que:

  • Los errores con hilos son más difíciles de reproducir porque el orden de ejecución cambia cada vez.
  • Para proteger zonas de código que no pueden ser usadas por dos hilos a la vez, se usa synchronized.

6.1 Pequeño ejemplo

public class Contador { private int valor = 0; public synchronized void incrementar() { valor++; } public int getValor() { return valor; } }

La palabra synchronized hace que solo un hilo pueda entrar en el método incrementar() cada vez, evitando que se pierdan incrementos.

7. Jikes y depuradores gráficos

Además de los IDEs, existen depuradores gráficos independientes, como el de Jikes. No es necesario que lo uses en tus primeros programas, pero es interesante saber que:

  • Puedes ver las variables en ventanas separadas.
  • Tienes una ventana para la pila de llamadas, otra para hilos, etc.
  • Funciona en varios sistemas operativos al estar escrito en Java.

En la práctica, lo más habitual en clase será usar el depurador integrado en el IDE, que ofrece una experiencia muy parecida.

8. JDB: el depurador de línea de comandos

Esta parte es para estudiantes que quieran profundizar un poco más y entender cómo funciona la depuración “por debajo”.

JDB es una herramienta del JDK para depurar desde la consola. Utiliza comandos de texto:

8.1 Ejemplo básico con JDB

Supón que tienes la clase HolaMundo:

public class HolaMundo { public static void main(String[] args) { String msg = "Hola"; System.out.println(msg); } }
  1. Compila con información de depuración:
    javac -g HolaMundo.java
  2. Inicia JDB:
    jdb HolaMundo
  3. Dentro de JDB, puedes hacer cosas como:
    • stop in HolaMundo.main: poner un breakpoint en main.
    • run: ejecutar el programa hasta el breakpoint.
    • print msg: ver el valor de msg.
    • step y next: avanzar por el código.

9. Ejercicios prácticos para clase

Ejercicio 1: Encuentra el bug

El siguiente código quiere calcular el área de un rectángulo, pero devuelve un resultado incorrecto. Usa mensajes en consola o el depurador del IDE para encontrar el fallo.

public static int areaRectangulo(int base, int altura) { int area = base + altura; // <-- Bug return area; } public static void main(String[] args) { int resultado = areaRectangulo(5, 3); System.out.println("Área = " + resultado); }
  • Pista 1: ¿Qué fórmula debería usarse?
  • Pista 2: Pon un breakpoint dentro de areaRectangulo.
  • Extensión: añade comprobaciones para valores negativos.

Ejercicio 2: Bucle infinito

Este programa se “queda pensando” y parece no terminar nunca. Usa el depurador para descubrir qué pasa.

public static void main(String[] args) { int i = 0; while (i < 10) { System.out.println("i = " + i); // i++; <-- Falta algo aquí } }
  • Pista 1: ¿Qué condición tiene el while?
  • Pista 2: Observa el valor de i durante la ejecución.
  • Pista 3: Corrige el código para evitar el bucle infinito.

Ejercicio 3: Depuración con condicionales

Este método debería devolver si un número es positivo, negativo o cero, pero no funciona bien para todos los casos.

public static String tipoNumero(int n) { if (n > 0) { return "positivo"; } else if (n > 0) { return "negativo"; } else { return "cero"; } }
  • Prueba con valores: 5, -3, 0.
  • Usa el depurador para seguir el flujo de los if.
  • Corrige las condiciones para que el método funcione bien.

Ejercicio 4: Diseña tus propios mensajes de depuración

Toma un programa que ya tengas (por ejemplo, uno que calcule notas, medias o tablas de multiplicar) y:

  1. Añade mensajes System.out.println() en los puntos clave.
  2. Intenta que los mensajes sean claros, por ejemplo: "[DEBUG] Entrando en calcularMedia con n = " + n.
  3. Ejecuta el programa y comprueba si el flujo coincide con lo esperado.

10. Glosario básico

Bug
Error o fallo en el programa que hace que no se comporte como se espera.
Depurar
Proceso de encontrar y corregir bugs en el código.
Breakpoint
Punto de ruptura: marca en una línea donde el programa se detiene al depurar.
Stack trace
Traza de pila: lista de métodos que se han ido llamando hasta llegar a un error.
Excepción
Evento que interrumpe el flujo normal del programa (por ejemplo, dividir entre cero, acceso a posición inválida de un array, etc.).
Hilo (thread)
Unidad de ejecución dentro de un programa. Varios hilos permiten hacer varias cosas “a la vez”.
JDB
Depurador de línea de comandos incluido en el JDK.
IDE
Entorno de Desarrollo Integrado (Eclipse, IntelliJ, NetBeans...), que incluye editor, compilador y depurador.
