Tareas para Probar el Debugger

Código Java para las tareas 
import java.util.ArrayList;
import java.util.List;

public class DebuggingExample {
    public static void main(String[] args) {
        List names = new ArrayList<>();
        names.add("Alice");
        names.add("Bob");
        names.add("Charlie");
        names.add("Diana");

        DebuggingExample example = new DebuggingExample();
        example.printNames(names);

        int totalLetters = example.calculateTotalLetters(names);
        System.out.println("Total de letras: " + totalLetters);

        double averageLetters = example.calculateAverageLetters(names, totalLetters);
        System.out.println("Promedio de letras por nombre: " + averageLetters);

        example.addName(names, "Edward");
        example.removeName(names, "Alice");
        example.printNames(names);
    }

    public void printNames(List names) {
        for (String name : names) {
            System.out.println(name);
        }
    }

    public void addName(List names, String nameToAdd) {
        names.add(nameToAdd);
        System.out.println("Added name: " + nameToAdd);
    }

    public boolean removeName(List names, String nameToRemove) {
        boolean result = names.remove(nameToRemove);
        System.out.println("Removed name: " + nameToRemove);
        return result;
    }

    public int calculateTotalLetters(List names) {
        int total = 0;
        for (String name : names) {
            total += name.length();
        }
        return total;
    }

    public double calculateAverageLetters(List names, int totalLetters) {
        return names.isEmpty() ? 0 : (double) totalLetters / names.size();
    }
}
1. Ejecutar el programa con un breakpoint inicial
  • Establece un breakpoint en la línea donde se llama al método example.printNames(names);.
  • Ejecuta el programa en modo depuración.
  • Observa las variables iniciales, especialmente la lista names, usando la ventana de variables.
2. Explorar printNames con Step Over
  • Usa Step Over para avanzar línea por línea dentro del método.
  • Observa cómo cambia el valor de name en cada iteración del bucle for.
  • Confirma que la lista names no se modifica.
3. Usar Step Into en calculateTotalLetters
  • Coloca un breakpoint en la línea donde se llama a example.calculateTotalLetters(names);.
  • Usa Step Into para entrar al método.
  • Observa cómo se acumula el valor de total en cada iteración del bucle.
4. Usar Step Into y Step Return en addName
  • Coloca un breakpoint en la línea donde se llama a example.addName(names, "Edward");.
  • Usa Step Into para entrar al método.
  • Observa cómo se modifica la lista names tras la operación names.add(nameToAdd);.
7. Verificar cambios en names
  • Inspecciona la lista antes y después de las llamadas a addName y removeName.
  • Confirma que los métodos modifican la lista correctamente.

Objetivo General

Estas tareas enseñarán a los estudiantes:

  • Cómo recorrer el código usando diferentes herramientas del debugger.
  • Cómo inspeccionar y modificar variables.
  • Cómo identificar errores lógicos o de ejecución.
  • Cómo evaluar expresiones en tiempo de ejecución para verificar cálculos.
