Tareas para Pruebas Unitarias con JUnit 5

Código Java para Pruebas Unitarias 
package com;

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculadoraTest {

    @Test
    void testSuma() {
        Calculadora calculadora = new Calculadora(5, 3);
        assertEquals(8, calculadora.suma(), "La suma debe ser correcta");
    }

    @Test
    void testResta() {
        Calculadora calculadora = new Calculadora(5, 3);
        assertEquals(2, calculadora.resta(), "La resta debe ser correcta");
    }

    @Test
    void testMultiplica() {
        Calculadora calculadora = new Calculadora(5, 3);
        assertEquals(15, calculadora.multiplica(), "La multiplicación debe ser correcta");
    }

    @Test
    void testDivide() {
        Calculadora calculadora = new Calculadora(6, 3);
        assertEquals(2, calculadora.divide(), "La división debe ser correcta");
    }

    @Test
    void testDivide0ThrowsException() {
        Calculadora calculadora = new Calculadora(5, 0);
        Exception exception = assertThrows(ArithmeticException.class, calculadora::divide0);
        assertEquals("División por 0", exception.getMessage());
    }
}
1. Pruebas Unitarias Básicas
  • Crea pruebas unitarias para validar los métodos suma, resta, multiplica y divide.
  • Usa assertEquals para comparar el resultado esperado con el obtenido.
  • Asegúrate de incluir mensajes descriptivos para identificar errores fácilmente.
2. Prueba de Manejo de Excepciones
  • Escribe una prueba para el método divide0 que verifique la excepción ArithmeticException.
  • Usa assertThrows para capturar la excepción lanzada.
  • Valida que el mensaje de la excepción sea "División por 0".
3. Configuración con @BeforeEach
  • Usa @BeforeEach para inicializar una instancia de Calculadora antes de cada prueba.
  • Verifica que cada método use esta instancia configurada.
4. Pruebas Parametrizadas con @ParameterizedTest
  • Implementa pruebas parametrizadas para el método suma usando diferentes conjuntos de datos.
  • Usa @CsvSource para definir entradas y resultados esperados.
  • Verifica que los valores proporcionados producen los resultados correctos.
5. Casos de Prueba para Valores Límite
  • Crea casos de prueba para números negativos, cero y valores extremos como Integer.MAX_VALUE.
  • Valida que los métodos de Calculadora manejen correctamente estos casos.

Objetivo General

Estas tareas ayudarán a los estudiantes a:

  • Escribir pruebas unitarias utilizando JUnit 5.
  • Validar métodos básicos y manejo de excepciones.
  • Implementar pruebas parametrizadas.
  • Probar valores límite para garantizar la robustez del código.
