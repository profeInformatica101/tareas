Tres en raya en consola (Java)

Guía docente para alumnado de FP • Funciones y tablas (arrays 2D) • Sin POO ni Swing

Java 11+ Consola Procedimental
Resultados de aprendizaje (RA)
  • Estructura programas en módulos/funciones con responsabilidades claras.
  • Utiliza arrays bidimensionales para representar información en cuadrícula.
  • Implementa validación de entradas y control de flujo (if, while, switch).
  • Trabaja con menús en bucle y lectura por Scanner.
  • Versiona el código con Git y realiza entregas en GitHub.
Prerrequisitos
  • Java instalado y ejecución por consola.
  • Manejo básico de for, while, if, switch.
  • Nociones de char[][] y Scanner.
  • Git configurado (usuario, correo y acceso a GitHub).
Secuencia didáctica

Objetivo: Crear TresEnRaya.java y preparar main.

  • Importa java.util.Scanner.
  • Deja preparado un esqueleto con funciones vacías: mostrarMenu, mostrarInstrucciones, jugar, inicializarTablero, mostrarTablero, hayGanador, tableroLleno.

Verificación: compila sin implementación.

Errores comunes: paquete/clase que no coincide con el nombre del archivo.

Objetivo: Controlar el flujo del programa con un menú iterativo.

  1. Implementa do { mostrarMenu(); opcion = sc.nextInt(); switch(opcion){...} } while(opcion != 3).
  2. Opciones: 1) jugar(sc), 2) mostrarInstrucciones(), 3) salir.
Valida entradas: si no es número, limpia el buffer y vuelve a pedir.

Verificación: El menú se repite hasta elegir 3; opciones fuera de rango muestran aviso.

Errores comunes: bucle infinito por no leer correctamente la opción.

Objetivo: Usar char[3][3] para representar el tablero.

  • inicializarTablero: rellena con espacio ' '.
  • mostrarTablero: imprime índices (0..2) y separadores.

Verificación: tablero vacío visible en consola.

Errores comunes: confundir filas y columnas al imprimir.

Objetivo: Alternar turnos y validar jugadas.

  1. Variable jugador inicia en 'X'.
  2. Bucle: pedir fila y columna, validar rango (0–2) y casilla libre.
  3. Colocar ficha y alternar jugador.
Verificación:
  • Imprime tablero tras cada jugada.
  • No permite sobreescribir casillas.
Errores comunes:
  • Olvidar limpiar el buffer de Scanner en entradas no numéricas.
  • Intercambiar índices: usar tablero[columna][fila] por error.

  • hayGanador: 3 iguales en fila, columna o diagonal.
  • tableroLleno: sin espacios libres → empate.

Verificación: prueba las 8 líneas ganadoras y un caso de empate.

Errores comunes: no comprobar ambas diagonales.

Checklist de pruebas (alumno)
  • El menú funciona y no se bloquea con opciones inválidas.
  • El tablero se muestra correctamente tras cada jugada.
  • No se permite jugar en casillas ocupadas.
  • Se detectan todas las líneas ganadoras (3 filas, 3 columnas, 2 diagonales).
  • Se detecta el empate cuando no quedan casillas.
  • El código compila sin warnings/errores y sigue una estructura por funciones.
