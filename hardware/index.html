Sistema de Procesamiento Central

El cerebro del computador

El sistema de procesamiento central es el componente principal de cualquier sistema de computador, a veces llamado el cerebro del computador. Es responsable por ejecutar instrucciones de software, ejecutar calculaciones matemáticas y lógicas, mantener la seguridad del sistema, detectar y manejar errores y verificar transferencias de datos entre componentes dentro del sistema, entre otros roles.

El modo fundamental de operación de un procesador es un proceso iterativo llamado ciclo de instrucción, también conocido como ciclo de ejecución del código Fetch. Este ciclo continuo es el mecanismo principal por el que un procesador ejecuta todas las instrucciones de software, desde el principio del computador y hasta el final de la instrucción.

Estas instrucciones pueden ser tan simples como añadir números, mover datos de un lugar a otro o verificar si un valor es cero.

Roles del CPU

  • Ejecutar instrucciones de software
  • Realizar cálculos matemáticos y lógicos
  • Mantener la seguridad del sistema
  • Detectar y manejar errores
  • Verificar transferencias de datos

Lenguaje nativo del CPU: ISA

La lengua nativa del CPU es definida por su arquitectura de instrucciones, o ISA, que incluye una lista predefinida de todas las instrucciones que un CPU específico puede entender. Los diferentes CPUs tienen diferentes ISAs.

X86 (CISC)

Por ejemplo, los procesadores Intel y AMD usan X86, que implementa una arquitectura de computadores con instrucciones complejas, también conocida como CISC.

ARM (RISC)

Los procesadores en los smartphones usan ARM, que implementa una arquitectura de computadores con instrucciones reducidas, también conocida como RISC.

Los procesadores modernos son increíblemente complejos. En el nivel fundamental, un procesador está formado por varias unidades que trabajan juntas para ejecutar instrucciones y realizar computaciones.

Componentes principales del procesador

Unidad Lógica Aritmética (ALU)

Maneja operaciones matemáticas y lógicas y es el motor computacional del procesador.

File de Registro

Guarda los datos que el procesador necesita para acceder rápidamente y que están en uso activo.

Unidad Load/Store (LSU)

Maneja las transferencias de datos entre el file de registro y los componentes de memoria.

Unidad de Control

Maneja el flujo de instrucciones y coordina todas las operaciones dentro del procesador.

A parte de estos cuatro componentes, el procesador también contiene varios subsistemas que manejan la memoria y la data, mantienen la seguridad y mejoran la performance del sistema.

Detalle de la ALU

El ALU es el motor computacional del procesador. Es responsable de ejecutar varios tipos de operaciones:

  • Operaciones aritméticas: adición, sustracción, multiplicación y división.
  • Operaciones lógicas: AND, OR, NOT, XOR.
  • Operaciones de comparación: igualdad, mayor, menor, etc.
  • Operaciones SHIFT: desplazan todos los bits a la izquierda o derecha.

El ALU recibe una instrucción y datos como entrada y produce un resultado. Además del resultado, también produce varias señales que ofrecen contexto:

  • Zero flag (resultado es cero)
  • Sign flag
  • Carry flag
  • Overflow flag
  • Otros indicadores de estado

Registros

Tenemos un set de unidades de memoria de alta velocidad llamadas registradoras. Su propósito es detener datos e instrucciones que están siendo usados activamente, ofreciendo al procesador acceso inmediato a la información relevante.

Propósito general

  • Detener datos para operaciones de ALU
  • Guardar resultados intermedios
  • Guardar direcciones de memoria

Propósito especial

  • Guardar la dirección de la siguiente instrucción
  • Guardar señales importantes del procesador

Registros de vector

Registradoras largas que permiten hacer una operación en múltiples elementos de datos al mismo tiempo.

Unidad de carga/guarda (LSU)

Una unidad de guarda de carga es una unidad de ejecución dentro del procesador, dedicada exclusivamente a manejar transferencias de datos entre el file de registro y la memoria externa.

La LSU puede ejecutar instrucciones de carga para recuperar datos de memoria en un registro e instrucciones de almacenamiento para escribir datos de registro de vuelta a la memoria.

Unidad de Control y reloj

La unidad de control contiene la lógica principal del procesador. Su trabajo es recoger instrucciones de memoria, decodificarlas y ejecutarlas mientras maneja todos los otros componentes.

Para asegurar que estos componentes se comuniquen y operen en sincronía, el procesador utiliza un clock que fija la velocidad. Genera una serie continua de pulsos eléctricos que coordinan todas las operaciones internas.

El rato de clock (velocidad) se mide en hertz, o en computadores modernos en gigahertz, lo que significa billones de ciclos por segundo. Un clock más alto permite completar más instrucciones por segundo.

Buses: la red de comunicación

Todos estos componentes se comunican usando una red compleja de líneas eléctricas internas llamadas buses. Estos buses transmiten datos, instrucciones y señales de control entre los diferentes componentes del procesador. Son la infraestructura de comunicación del CPU.

El ciclo de instrucción (5 etapas clásicas)

Los componentes del procesador trabajan juntos para ejecutar un programa a través de un proceso continuo llamado ciclo de instrucciones. Un ciclo clásico incluye cinco etapas principales:

  1. Fetch (búsqueda): la unidad de control mira el contador de programa, que guarda la dirección de memoria de la siguiente instrucción, la trae a un registro llamado registro de instrucciones y luego incrementa el contador de programa.
  2. Decode (decodificación): la unidad de control toma la instrucción y la decodifica, descubriendo qué operación se debe hacer y qué registros o direcciones de memoria están implicadas (por ejemplo, ADD R1 R2).
  3. Execute (ejecución): el procesador ejecuta la operación especificada. Las operaciones aritméticas o lógicas las realiza la ALU; el movimiento de datos lo inicia la unidad de carga/guarda; una instrucción de salto condicional puede cambiar el contador de programa.
  4. Memory (acceso a memoria): si la instrucción lo requiere, se accede a memoria para leer o escribir datos.
  5. Write back (escritura de resultados): se guarda el resultado final en el registro o la ubicación de memoria correspondiente.

Las dos últimas etapas (memoria y escritura) son opcionales y sólo ocurren cuando la instrucción lo requiere.
