Ejercicios Thymeleaf con Spring Boot

De básico a avanzado, usando Thymeleaf y Spring Boot.

Repositorio para hacer fork

El punto de partida de todos los ejercicios es el repositorio base:

https://github.com/profeInformatica101/SpringBootApp

Tras hacer fork, tu repositorio quedará como: https://github.com/<tu_usuario>/SpringBootApp

A partir de main crearás ramas por ejercicio y, cuando estén terminadas, se irán fusionando en una rama final de integración.

Ramas de trabajo

  • main – Código base del proyecto (no tocar).
  • final-thymeleaf – Rama final donde se van mergeando todos los ejercicios.
  • ejercicios-thymeleaf – (Opcional) Rama de apoyo para ir probando en clase.
  • soluciones-thymeleaf – Posibles soluciones de referencia (del profesor).
  • formularios-thymeleaf – Ejemplos de formularios con envío y procesado.
  • Cada ejercicio se hace en su propia rama (por ejemplo: ej01-hola-thymeleaf, ej02-tabla-productos, ..., ej16-select-etiquetas) y luego se fusiona a final-thymeleaf mediante merge o Pull Request.

Crea una vista Thymeleaf que muestre un mensaje de bienvenida y el nombre de un usuario enviado desde el controlador. El mensaje debe aparecer dentro de un <h1> y el nombre del usuario dentro de un <span>.

  • Usa th:text para reemplazar el contenido de una etiqueta HTML: th:text="${mensaje}".
  • En el controlador, añade atributos al modelo con model.addAttribute("nombre", "Ana").
  • En la plantilla, recupera el atributo con ${nombre} dentro de th:text.

  1. Crea un método en tu controlador que devuelva un String con el nombre de la vista.
  2. En ese método, añade mensaje y nombre al Model.
  3. En la vista, usa <h1 th:text="${mensaje}">...</h1> y <span th:text="${nombre}">...</span>.

  • Rama de trabajo: ej01-hola-thymeleaf
  • Crear a partir de main:
    git checkout main
    git checkout -b ej01-hola-thymeleaf
  • Tras terminar, subir la rama:
    git push origin ej01-hola-thymeleaf
  • Fusionar en final-thymeleaf mediante merge o Pull Request.

Crea una tabla que muestre una lista de productos (nombre, precio, categoría) que recibes desde el controlador como una lista de objetos. Cada fila debe representar un producto.

  • Utiliza th:each para iterar: <tr th:each="producto : ${listaProductos}">.
  • Dentro de la fila, usa th:text para mostrar atributos: th:text="${producto.nombre}", th:text="${producto.precio}".
  • En el controlador devuelve una List<Producto> en el modelo.

  1. Crea la clase Producto con nombre, precio, categoria.
  2. En el controlador, crea una lista de objetos y pásala al Model como listaProductos.
  3. En la vista, crea la tabla con encabezados y usa th:each en el <tr> para generar las filas.

Diseña una vista que muestre un mensaje de alerta si la lista de productos está vacía y, en caso contrario, muestre la tabla de productos.

  • Usa th:if para mostrar contenido solo cuando se cumpla una condición: th:if="${listaProductos.size() == 0}".
  • Usa th:unless para la condición contraria: th:unless="${listaProductos.isEmpty()}".
  • Combina th:if con componentes de Bootstrap como .alert para mostrar mensajes visuales.

  1. En el controlador, pasa siempre listaProductos (aunque esté vacía).
  2. En la vista, crea un bloque <div class="alert alert-warning"> con th:if si la lista está vacía.
  3. Envuelve la tabla de productos en un div con th:unless para mostrarla solo cuando haya elementos.

Añade a cada fila de la tabla de productos un botón "Ver detalle" que lleve a una página de detalle del producto usando su id en la URL.

  • Usa th:href para generar URLs dinámicas: th:href="@{/productos/{id}(id=${producto.id})}".
  • El patrón @{} permite construir rutas usando path variables y parámetros.
  • Combina esto con un botón Bootstrap: <a class="btn btn-sm btn-outline-primary" ...>.

  1. Define una ruta en el controlador como @GetMapping("/productos/{id}").
  2. En el método, busca el producto por su id y pásalo al modelo.
  3. En la tabla, añade una última columna con un enlace que use th:href y la expresión @{...} para construir la URL.

Crea un formulario para dar de alta un producto con campos: nombre, precio y categoría. El formulario debe estar vinculado a un objeto del modelo y debe enviarse a un endpoint POST.

  • Usa th:object en el formulario: <form th:object="${producto}" ...>.
  • Vincula los campos con th:field: <input th:field="*{nombre}" />.
  • La expresión *{...} se refiere al objeto definido en th:object.

  1. En la ruta GET de alta, añade al modelo un nuevo Producto() como producto.
  2. En la vista, declara th:object="${producto}" en el formulario.
  3. Define th:field para cada campo: nombre, precio y categoría.
  4. Crea una ruta @PostMapping para recibir el objeto completado y guardarlo.

Añade validación al formulario de producto (por ejemplo: nombre obligatorio, precio > 0) y muestra mensajes de error junto a cada campo cuando haya errores de validación.

  • Usa th:errors para mostrar mensajes: <div th:if="${#fields.hasErrors('nombre')}" th:errors="*{nombre}"></div>.
  • La utilidad #fields permite comprobar si un campo tiene errores.
  • Decora los mensajes con class="text-danger small" para que encajen con Bootstrap.

  1. Anota la clase Producto con validaciones (@NotBlank, @Positive, etc.).
  2. En el @PostMapping, usa @Valid y BindingResult.
  3. Si hay errores, devuelve la misma vista del formulario con esos errores.
  4. En la vista, añade bloques th:errors junto a cada campo.

Añade al producto un campo fechaAlta y muestra el precio y la fecha con formato amigable: precio con dos decimales y símbolo €, y fecha en formato dd/MM/yyyy.

  1. Añade el atributo fechaAlta en la clase Producto.
  2. Inicializa la fecha en el controlador (por ejemplo, LocalDate.now()).
  3. Muestra en la tabla una nueva columna con la fecha formateada y el precio formateado.

Internacionaliza la vista de listado de productos para que los textos de la interfaz se muestren en español o inglés según el parámetro lang de la URL.

  • Crea ficheros messages.properties, messages_es.properties, messages_en.properties.
  • Define claves como titulo.lista.productos=Listado de productos.
  • En la plantilla, usa th:text="#{titulo.lista.productos}" en lugar de texto literal.
  • Añade enlaces para cambiar de idioma: th:href="@{/productos(lista=${listaProductos}, lang='es')}" y lang='en'.

  1. Configura en Spring Boot el MessageSource si no lo tienes ya definido.
  2. Asegúrate de que el LocaleResolver tenga en cuenta el parámetro lang.
  3. Sustituye los textos fijos de la vista por claves de mensajes #{...}.
  4. Comprueba que al cambiar ?lang=es / ?lang=en se actualiza el idioma de la interfaz.

Crea un formulario de producto que incluya un campo <select> para elegir la categoría del producto. Las opciones de la lista se deben obtener desde el modelo (por ejemplo, una lista de cadenas con "Tecnología", "Hogar", "Deporte", "Ropa", etc.).

Al enviar el formulario, el controlador debe recibir el producto con su categoría seleccionada y mostrar una vista de resumen con el nombre del producto y la categoría elegida.

  • En el controlador, añade al modelo una lista de categorías: model.addAttribute("categorias", List.of("Tecnología","Hogar","Deporte"));
  • En el formulario, vincula el objeto producto: <form th:object="${producto}" ...>
  • Define el <select> con th:field:
    <select th:field="*{categoria}" class="form-select">
  • Dentro del <select> itera con th:each:
    <option th:each="cat : ${categorias}"
    th:value="${cat}"
    th:text="${cat}"></option>
  • Puedes añadir una opción por defecto:
    <option value="" disabled selected>-- Elige categoría --</option>

  1. Crea o reutiliza la clase Producto con el atributo categoria.
  2. En el @GetMapping, inicializa Producto y la lista de categorías, añadiéndolos al modelo.
  3. En la vista del formulario, usa th:object y el <select> con th:field="*{categoria}".
  4. En el @PostMapping, recibe el objeto Producto y muestra una vista de resumen con el nombre y la categoría.

  • Rama de trabajo: ej15-select-categoria
  • Crea la rama a partir de main:

Extiende el concepto anterior creando un formulario que permita seleccionar múltiples etiquetas para un producto (por ejemplo: "Oferta", "Nuevo", "Reacondicionado", "Destacado", "Outlet") usando un <select multiple>.

El formulario debe enviar un objeto que contenga una lista de etiquetas seleccionadas. La vista de resultado debe mostrar el nombre del producto y un listado de todas las etiquetas escogidas.

  • Crea en tu modelo un atributo tipo lista, por ejemplo:
    private List<String> etiquetas;
  • En el controlador, pasa una lista de opciones posibles:
    model.addAttribute("opcionesEtiquetas", List.of("Oferta","Nuevo","Reacondicionado","Destacado","Outlet"));
  • En la plantilla, define el <select> con multiple:
    <select multiple th:field="*{etiquetas}" class="form-select">
  • Dentro del <select>, recorre las opciones:
    <option th:each="et : ${opcionesEtiquetas}"
    th:value="${et}"
    th:text="${et}"></option>
  • Thymeleaf se encargará de convertir las selecciones en una lista de String en el objeto de formulario.

  1. Crea una clase de formulario, por ejemplo ProductoEtiquetasForm, con nombreProducto y List<String> etiquetas.
  2. En el @GetMapping, inicializa el formulario y la lista opcionesEtiquetas y añádelos al modelo.
  3. En la vista, utiliza th:object="${productoEtiquetas}" y el <select multiple>.
  4. En el @PostMapping, recibe el formulario y muestra en una vista de resumen el nombre del producto y todas las etiquetas seleccionadas (por ejemplo, en una lista con badges de Bootstrap).

  • Rama de trabajo: ej16-select-etiquetas
  • Crea la rama a partir de main (o desde la rama anterior si lo prefieres):

    git checkout -b ej16-select-etiquetas
  • Implementa el formulario con select multiple y la vista de resumen.
  • Haz commit con un mensaje como:
    "Ejercicio 16 - Select múltiple de etiquetas"
  • Sube los cambios:
    git push origin ej16-select-etiquetas
  • Fusiona en final-thymeleaf para integrar con el resto de ejercicios.
