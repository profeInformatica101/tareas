Práctica: Diseño de Interfaz Web con Pencil

Este proyecto consiste en desarrollar una plataforma web que permita a los estudiantes de un ciclo formativo superior en desarrollo de aplicaciones web mostrar sus proyectos finales. La plataforma debe permitir la autenticación de usuarios, la subida y gestión de proyectos, así como la visualización y búsqueda de estos proyectos por parte de otros estudiantes, profesores y empresas interesadas. Utiliza esta actividad para demostrar tus habilidades de desarrollo y colaboración en un entorno de trabajo real y práctico.

En esta práctica no se programará ninguna funcionalidad. El objetivo es realizar un diseño visual de una aplicación web usando la herramienta Pencil Project.

El sistema a diseñar es una plataforma que permitiría a estudiantes publicar proyectos, y a otros usuarios (profesores, empresas, compañeros) visualizar dichos proyectos. Aunque no se implementará, deberás pensar cómo funcionaría la aplicación y representarlo visualmente.

¿Qué debe hacer la aplicación? (Requisitos para el diseño)

  • Inicio de sesión y registro de usuarios.
  • Panel personal del estudiante donde pueda subir y gestionar proyectos.
  • Vista pública de los proyectos con buscador.
  • Página de detalle para cada proyecto.

Entregables obligatorios

  1. Prototipo visual de la interfaz realizado con Pencil, incluyendo al menos las siguientes pantallas:
    • Inicio de sesión
    • Registro de usuario
    • Panel de usuario
    • Listado de proyectos
    • Vista de detalle del proyecto

    Debe exportarse en formato PDF o PNG.

  2. Diagrama de Casos de Uso que represente los actores (usuario registrado, visitante) y sus interacciones con el sistema.

Entregable opcional

Como mejora voluntaria puedes incluir:

  • Diagrama de Clases básico con las entidades que tendría el sistema.

Pasos recomendados

  1. Comprende bien la idea general de la plataforma.
  2. Dibuja un boceto en papel con las pantallas necesarias.
  3. Descarga e instala Pencil Project.
  4. Crea el prototipo de las pantallas indicadas.
  5. Exporta el prototipo y el diagrama de casos de uso.
  6. Sube los archivos a Moodle o donde se indique.
Esta práctica tiene como objetivo que entiendas el diseño visual de una interfaz y cómo se representan las funcionalidades mediante casos de uso. No se requiere programación.
